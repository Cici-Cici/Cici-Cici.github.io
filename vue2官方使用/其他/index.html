定义

组件

组件是可复用的 Vue 实例，且带有一个名字，它们与 new Vue 接收相同的选项，例如 data、computed等

vue应用

通过 new Vue 创建的根 Vue 实例，以及可选的嵌套的、可复用的组件树组成；

指令

指令带有前缀 v-，以表示它们是 Vue 提供的特殊 attribute。当表达式的值改变时，将其产生的连带影响，响应式地作用于 DOM

声明式渲染（文本）

允许使用简洁的模版语法来将数据渲染进DOM系统

MVVM模式

是Model-View-ViewModel的简写，model是数据，view是用户界面，ViewModel可以取出 Model 的数据同时帮忙处理 View 中由于需要展示内容而涉及的业务逻辑。ViewModel将视图 UI 和业务逻辑分开，把Model和View关联起来，并实现双向数据绑定
MVVM核心思想，是关注model的变化，让MVVM框架利用自己的机制自动更新DOM

事件修饰符

1.事件修饰符对事件进行约束等
2.键盘事件可添加按键修饰符（下面代码按enter建触发），以前使用的按键码大部分可被别名代替，可以自定义按键别名
3.系统修饰符，按键组合及其问题（.exact 修饰解决）实现各种情况下的时间触发、
4.有鼠标按键修饰符

其他

  • data 必须是一个函数
    返回的是变量的话，使用多次同一个组件间会互相影响，因为用的同一个数据。但是如果是函数的话，用的是同一个函数，不会影响，每次返回变量都是一个新的拷贝
  • 不要在选项 property 或回调上使用箭头函数，比如

created: () => console.log(this.a) 或vm.$watch('a', newValue => this.myMethod())
因为箭头函数并没有 this，this 会作为变量一直向上级词法作用域查找

  • 采用BEM规范书写css能增加代码的阅读性和可维护性。

  • v-if，和v-for在同一个组件上的问题（可以这样但不推荐）
    不建议这样做是因为效率不好，渲染几个特定元素是没有必要去遍历不相关元素的。而放一个组件上不符合这个基准，因为v-for的优先级更高，不管需不需要渲染都会去遍历

  • v-for需要key值；用v-if，和v-else切换组件时，如果这两个组件内部有相同的元素，会发现这个相同的元素不会销毁了又创建
    原因：Vue 会尽可能高效地渲染元素，通常会复用已有元素而不是从头开始渲染。这么做除了使 Vue 变得非常快之外，还有其它一些好处，向input元素，在这样一种场景，不想让用户重新输入内容的话就不需要重复创建该元素。
    key值：(是 Vue 识别节点的一个通用机制)其他场景需要重复的时候，就要用一个具有唯一值的 key attribute 去给 Vue 一个提示，以便它能跟踪每个节点的身份，从而重用和重新排序现有元素

  • 绑定内联样式自动添加前缀，多重值<div :style="{ display: ['-webkit-box', '-ms-flexbox', 'flex'] }"></div>

  • 异步组件

  • AB组件间互相引用的解决方案

  • 极少数情况下可能需要手动强制更新
