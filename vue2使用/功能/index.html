默认

  • Vue 会尽可能高效地渲染元素，通常会复用已有元素而不是从头开始渲染。用key值可以防止复用
  • 在将 v-bind 用于 class 和 style 时，Vue.js 做了专门的增强。表达式结果的类型除了字符串之外，还可以是对象或数组。
  • 所有的数据绑定，Vue.js 都提供了完全的 JavaScript 表达式支持，只能包含单个表达式（模版表达式中不能访问用户定义的全局变量）
  • 当实例创建时就已经存在于 data 中的 property 是响应式的，当这些数据改变时，视图会进行重渲染；如果你添加一个新的 property，改动将不会触发任何视图的更新
  • vue将被侦听的数组的变更方法进行了包裹，所以它们也将会触发视图更新。
  • 由于 JavaScript 的限制，Vue 不能检测数组和对象的变化（可以采用变更方法来变更数组，对于非变更方法可以替换数组来达到相应式效果）
    1.对于对象：Vue 无法检测 property 的添加或移除。由于 Vue 会在初始化实例时对property 执行 getter/setter 转化，所以 property 必须在 data 对象上存在才能让 Vue 将它转换为响应式的
    2.对于数组：
    当你利用索引直接设置一个数组项时，例如：vm.items[indexOfItem] = newValue
    当你修改数组的长度时，例如：vm.items.length = newLength
  • 自定义组件 会被作为无效的内容提升到外部（因为有些 HTML 元素，对于哪些元素可以出现在其内部是有严格限制的），并导致最终渲染结果出错。
    解决办法：is=”blog-post-row”
