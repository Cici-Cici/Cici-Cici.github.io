赖继娟 2024-04-24 17:12:26

v-if vs v-show

  1. v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销
  2. v-if 是“真正”的条件渲染，因为它会确保在切换过程中条件块内的事件监听器和子组件适当地被销毁和重建;v-show 只是简单地切换元素的 CSS property display

计算属性和方法

  1. 计算属性是基于响应式依赖进行缓存的（每当触发重新渲染时，相关响应式依赖发生改变时它们才会重新求值）
    只要 message 还没有发生改变，多次访问 reversedMessage 计算属性会立即返回之前的计算结果，而不必再次执行函数
  2. 调用方法将总会再次执行函数

计算属性和侦听器

  1. 计算属性在大多数情况下更合适，执行异步操作 (访问一个 API)，这是计算属性无法做到的

class和style绑定中使用对象语法和数组语法

class

  1. 用一个变量来控制使用哪个样式
  2. 可以和 class=”static”并用
  3. 可以不必内联
  4. 数组语法：应用一个class列表

选择

  1. 如果只是想用变量控制所有样式是否生效
  2. 如果想用变量控制样式是否生效，还要加上一定生效的样式
  3. 如果这些样式中有的样式需要变量控制，有的样式需要计算或者改变
  4. 如果需要应用多个样式，样式都需要被计算
